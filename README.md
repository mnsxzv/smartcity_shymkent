# 🏙️ Smart City Shymkent — ИИ Мониторинг Инцидентов

Интеллектуальная система мониторинга городских проблем для города Шымкент на базе искусственного интеллекта и гибридного вычисления геокоординат. Проект разработан в рамках хакатона.

---

## 🚀 О проекте

Система автоматизирует сбор жалоб от жителей через **Telegram-бота**, анализирует текст с помощью больших языковых моделей (**LLM через Groq API**), автоматически определяет категорию проблемы и вычисляет точные географические координаты для отображения на интерактивной веб-панели в режиме реального времени.

### ✨ Ключевые фичи:
* **Интеллектуальный парсинг:** Жителю не нужно заполнять формы — ИИ сам понимает контекст, адрес и суть проблемы из свободного текста.
* **Гибридный ИИ-Математический калькулятор координат:** Нейросеть рассчитывает смещение номеров домов вдоль главных координатных осей Шымкента (Кунаева, Тауке хана, Рыскулова, Байдибек би).
* **100% Отказоустойчивость (Fallback-система):** В бэкенд встроен жесткий перехватчик триггеров и ошибок. Если API ИИ недоступно, система мгновенно переходит на автономный режим, сохраняя работоспособность карты.
* **Премиальный UX/UI:** Интерактивная карта на базе Leaflet.js с кинематографичной анимацией подлёта (`flyTo`) и центрированием маркера при клике на инцидент. Дизайн панели выполнен в современном стиле **Glassmorphism**.

---

## 🛠️ Технологический стек

* **Backend:** Python 3.10+, Flask (API для фронтенда), Aiogram 3.x (Асинхронный Телеграм-бот), SQLite3 (Легковесная СУБД).
* **AI Модуль:** Groq API (Модель `mixtral-8x7b-32768`), Prompt Engineering, Hybrid Keyword-LLM routing.
* **Frontend:** HTML5, CSS3 (Glassmorphic дизайн), JavaScript (ES6+), Leaflet.js (Интерактивные карты) + CartoDB Dark Matter (Темизация).

---

## 📦 Архитектура системы

1. **User** ➔ Отправляет жалобу («На Кунаева возле ЦУМа перегорел фонарь») в Telegram.
2. **Telegram Bot (main.py)** ➔ Перехватывает текст и отправляет в ИИ-анализатор.
3. **AI Pipeline** ➔ Проверяет жесткие триггеры ➔ Запрашивает Mixtral 70B ➔ Выдает валидный JSON.
4. **Database** ➔ Запись сохраняется в `city_issues.db` с координатами и категорией.
5. **Frontend (index.html)** ➔ Каждые 3 секунды опрашивает Flask API (`/api/issues`) и динамически обновляет маркеры на карте без перезагрузки страницы.
