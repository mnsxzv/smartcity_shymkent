# 🏙️ Ситуационный Центр Мониторинга Инцидентов (Шымкент)

Интеллектуальная система интеллектуального анализа и визуализации городских проблем в режиме реального времени. Проект связывает Telegram-бота для сбора жалоб от жителей, ИИ-аналитику для распознавания локаций и интерактивную веб-карту.

---

## 🚀 Как это работает (Архитектура)

1. **Сбор данных:** Житель города отправляет текстовое сообщение о проблеме (например, *"На Самал-3 авария, прорвало трубу"*) в Telegram-бот.
2. **ИИ-Парсинг и Геокодирование:** Бэкенд на Python перехватывает сообщение. Скрипт использует алгоритм сортировки весов локаций (для исключения конфликтов вида *Самал-3* / *3-й микрорайон*) и обращается к LLM (Mixtral-8x7b через Groq API) для определения категории, адреса и радиуса аварии.
3. **База данных:** Инцидент сохраняется в локальную БД SQLite3 со статусом `Новая`.
4. **API Сервер:** Flask-приложение раздает накопленные маркеры через REST API (`/api/issues`).
5. **Веб-интерфейс:** Фронтенд (HTML/CSS/JS + Leaflet.js), опубликованный на GitHub Pages, каждые 4 секунды опрашивает API и динамически выводит неоновые буферные зоны на темную карту Картографии.

---

## ⚡ Особенности и Инновации

* **Умный алгоритм детекции локаций:** Исключены ложные срабатывания регулярных выражений. Система автоматически приоритизирует составные жилые массивы перед одиночными номерными микрорайонами.
* **Динамический ИИ-Индикатор статуса (LIVE):** На фронтенде реализован интерактивный виджет. Он автоматически пингует сервер: горит зеленым (`ГОТОВ`), если связь с ИИ стабильна, и переключается в красный режим (`ОФЛАЙН`), если туннель или бэкенд недоступны.
* **Туннелирование для Mobile:** Благодаря интеграции заголовков обхода предупреждений (`ngrok-skip-browser-warning`), система полностью отзывчива и работает на мобильных устройствах через удаленные туннели.
* **Адаптивный интерфейс:** Удобный Sidebar переключается в мобильный режим (split-экран сверху/снизу) при открытии с телефона.

---

## 🛠️ Технологический стек

* **Frontend:** HTML5, CSS3 (Flexbox/Grid, Анимации), JavaScript (ES6), Leaflet.js (Картография).
* **Backend:** Python 3.x, Flask (REST API), Flask-CORS.
* **Telegram Bot:** Aiogram 3.x (Асинхронный движок).
* **ИИ-Модуль:** Groq API / OpenAI SDK (Модель `mixtral-8x7b-32768`).
* **База данных:** SQLite3.
* **Туннелирование:** Ngrok (для связи GitHub Pages с локальным хостом).

## 💻 Локальное развертывание

### 1. Подготовка окружения
Создайте в корневой папке проекта файл с именем `.env` и вставьте в него ваши ключи (без пробелов вокруг знака равно):
BOT_TOKEN=твой_токен_от_botfather
GROQ_API_KEY=твой_ключ_от_groq_api

### 2. Установка зависимостей и запуск системы
Библиотеки Flask и Flask-CORS устанавливать вручную не нужно. Логика скрипта автоматически проверит их наличие и доустановит через pip при первом старте. 

Запустите главный файл проекта через терминал:
python main.py

После этого в консоли должно появиться сообщение: "🚀 ИИ-сервер запущен! Ждем сообщения..." — это подтверждает, что и асинхронный Telegram-бот, и API-сервер Flask успешно работают параллельно на порту 5000.

### 3. Настройка доступа для мобильных устройств
Так как веб-сайт на GitHub Pages работает через защищенный протокол HTTPS, браузер телефона заблокирует прямые запросы к незащищенному localhost вашего ноутбука. Для обхода этой защиты необходимо поднять безопасный туннель:

1. Запустите Ngrok на порт Flask-сервера:
   ngrok http 5000
2. Скопируйте сгенерированную HTTPS-ссылку (формата https://xxxx.ngrok-free.dev).
3. Откройте файл фронтенда `index.html`, найдите строку 172 и обновите переменную API_URL:
   const API_URL = 'https://xxxx.ngrok-free.app';
4. Сохраните файл и отправьте изменения на GitHub через `git push`. Теперь мобильный телефон сможет отправлять запросы на ваш локальный сервер из любой точки мира.
